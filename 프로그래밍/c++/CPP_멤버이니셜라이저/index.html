    [C++] Member Initializer : 설명

    빠른 링크 (Link!)

    • 서론
    • 1. 사용방법
    • 2. 멤버변수 초기화
    • 3. 부모 멤버변수 초기화
    • 4. 상수 멤버변수 초기화 (< C++11)
    • 5. 레퍼런스 멤버변수 초기화
    • 6. 쓸모없는 결론

    서론

    멤버변수의 선언 후 기본값이 Visual Studio에서는
    디버그 환경으로 빌드할 시 0이 아닌 0xCDCDCD라는 이상한 값이 들어가 있습니다.
    기본값이 잘못되어 런타임에서 프로그램이 실행되다가 갑자기 멈춘다거나
    결과물이 다르게 출력되는 현상이 나타납니다.

    값을 초기화 한다는 것은 프로그래밍에서 매우 중요한 역할을 합니다.
    클래스 멤버변수를 초기화하려면 생성자를 이용하거나 멤버이니셜라이저를 이용합니다.
    레퍼런스 타입을 가진 멤버변수 같은 경우는 생성자를 이용해 초기화할 수 없습니다.
    멤버이니셜라이저의 호출은 객체가 생성되기 전에 호출됩니다.

    • 멤버이니셜라이저가 사용되는 방식은 크게 4가지정도 있습니다.
      1. 멤버변수 초기화
      2. 부모 멤버변수 초기화
      3. 상수 멤버변수 초기화 (< C++11)
      4. 레퍼런스 멤버변수 초기화

    이번 글에서는 멤버이니셜라이저를 이용하여
    멤버변수 초기화 방법에 대해 알아봅니다.
    기본적인 방법만 알고 있으면 크게 어려울 것이 없습니다.

    1. 사용방법

    2. 멤버변수 초기화

    출력 결과:
	* 멤버 statusID_ 값이 5로, Name_ 값이 'Chris' 로 초기화 되었다.

Parent:: ID = 5, Name = Chris

    3. 부모 멤버변수 초기화

    자식(Child) 객체의 멤버이니셜라이저를 이용해 부모(Parent) 객체의 멤버변수들을 초기화할 수 있습니다. 멤버이니셜라이저에서 가장 많이 사용되는 방법이라고 생각합니다.

    출력 결과:
	* 자식(Child) 객체의 멤버이니셜라이저에서 부모(Parent) 매개변수 생성자를 호출하여
	* 부모(Parent) 멤버변수가 초기화 되었다.

Child:: ID = 10, Name = James
Parent:: ID = 99, Name = Jack

    4. 상수 멤버변수 초기화 (< C++11)

    C++11 에서는 멤버이니셜라이저를 사용하지 않고도 상수 멤버변수를 선언하자마자 쉽게 초기화 할 수 있습니다.

    (warning: in-class initialization of non-static data member is a C++11 extension [-Wc++11-extensions])

    GCC 표준 옵션(-std=98)을 적용하여 컴파일 해봤지만… 위와 같은 warning만 보여지고 error는 없었습니다.

    5. 레퍼런스 멤버변수 초기화

    레퍼런스의 특징은 선언을 했으면 반드시 초기화 값을 가지고 있어야합니다.
    nullptr값으로 변경할 수 없으며, nullptr은 상수 레퍼런스(const &ref)만 사용가능합니다.

    레퍼런스 타입 멤버변수를 가진 객체를 생성하기 위해서는
    레퍼런스 타입 멤버변수는 멤버이니셜라이저를 이용하여 반드시 초기화가 되어야만 합니다.

    클래스에서 여러개의 생성자를 가지고 있으면
    모든 생성자에서 멤버이니셜라이저를 이용하여 레퍼런스 타입의 멤버변수를 초기화해야만 합니다.

    (error: constructor for ‘Point’ must explicitly initialize the reference member ‘x’)

    멤버이니셜라이저로 초기화 하지 않으면 위와 같은 컴파일 오류가 발생하게 됩니다.

    출력 결과:
	* main 함수에서 x, y를 레퍼런스 타입으로 받아 객체에 넘겨주었다.

Point:: x = 10, y = 20

    6. 쓸모없는 결론

    총 4가지 사용방법에 대해 코드와 함께 정리해보았습니다.
    기본적으로 멤버변수를 초기화할 때에는 멤버이니셜라이저를 사용하는 것이 좋습니다.
    멤버이니셜라이저는 객체가 생성되기 전에 호출됩니다.
    부모 생성자를 초기화할 때에는 아주 유용할 것 같습니다.
    상수 멤버변수 초기화는 C++11부터는 편하게 초기화할 수 있으므로 사용되지 않습니다.
