    [MFC] CFileDialog : 여러개의 파일을 선택하여 정보 가져오기

    빠른 링크 (Link!)

    • 서론
    • 1. 기존 버퍼를 교체
    • 2. 버퍼의 크기 변경
    • 3. 다중 선택 가능한 파일 다이얼로그 띄우기
    • 4. 여러개의 파일 정보 가져오기
    • 4. 결론

    서론

    FileDialog에서는 여러개의 파일을 선택하는 기능을 지원하고 있습니다.
    Adobe사 포토샵 같은 프로그램을 보면 이미지를 여러개 선택해 불러와서 사용자 작업을 처리합니다.

    위와 같은 작업은 개발자가 프로그램을 만들 때
    여러개의 파일을 선택해서 정보를 얻어 처리하는 프로그래밍을 했기에 가능합니다.

    이 글에서는 여러개의 파일을 선택해 선택된 파일에 대한 정보를 가져오는 방법에 대해 적어보겠습니다.

    선택한 여러개의 파일 정보를 구하기 위해서는 OFN_ALLOWMULTISELECT 플래그가 적용되어야 합니다.
    FileDialog는 반드시 열기모드(TRUE)로 작동해야합니다.

    [ ! ] 아래 작업은 DoModal() 함수가 호출되기 이전에 진행해야 합니다.

    • 여러개의 파일을 선택하기 위해서는 아래에 적힌 작업을 모두 진행해야 합니다.
      1. 기존 버퍼를 교체 (새롭게 버퍼를 만듬)
      2. FileDialog가 가지고 있는 버퍼 크기 변경

    1. 기존 버퍼를 교체

    FileDialog가 여러개의 파일 정보를 얻기 위해서는 큰 버퍼가 필요합니다.

    기본적으로 버퍼를 가지고 있지만 1개의 파일에 대한 PATH 정보를 담을 수 있으며
    기본 크기는 260으로 설정되어 있습니다.

    작은 크기의 버퍼를 사용할 경우 여러개의 파일을 선택했을 때 오류가 발생하거나
    선택한 모든 파일에 대한 일부 파일의 정보만 가져올 수 있습니다.

    기존 버퍼를 변경하기 위해서는 버퍼의 위치(포인터)를 나타내는 멤버변수 m_ofn.lpstrFile 값을 변경해야 합니다.
    CString 지역변수를 선언하고 가르키는 버퍼를 변경해보도록 하겠습니다.
    CString 클래스 GetBuffer() 함수를 이용해서 해당 객체가 가지고 있는 데이터의 시작 위치(포인터)를 얻어올 수 있습니다.
    GetBuffer() 함수를 이용해 새롭게 객체를 할당 받았다면
    반드시 ReleaseBuffer() 함수를 호출하여 메모리를 해제해야 됩니다.

    버퍼를 변경하는 코드는 다음과 같습니다.

    버퍼만 변경한다고 해서 프로그램이 정상적으로 작동하는 것은 아닙니다.
    고정된 버퍼의 크기는 런타임 오류가 발생할 위험이 있습니다.

    사용자가 최대 몇개의 파일을 선택할 것인지에 대해 생각하여 버퍼의 크기를 명시하는 것이 좋습니다.

    2. 버퍼의 크기 변경

    버퍼의 크기를 변경하기 위해서는 멤버변수 m_ofn.nMaxFile 값을 변경해야 합니다.
    버퍼의 크기를 지정할 때에는 주어진 공식을 이용하는 것이 좋습니다.
    버퍼의 크기 계산 공식은 n * (MAX_PATH + 1) + 1) 입니다. (참고:MSDN)
    이 공식은 n개의 파일에 대한 PATH 정보를 담을 수 있는 것을 의미합니다.

    최소 99개의 파일을 선택할 수 있는 버퍼 크기를 변경하는 코드는 다음과 같습니다.

    선택된 파일에 따라 PATH 정보 크기는 다릅니다.
    그러므로 PATH 정보가 차지하고 있는 데이터의 크기는 항상 다릅니다.
    PATH가 아무리 긴다고 가정하에 99개의 파일 정보를 가져올 수 있으며
    반대로 PATH가 너무 짧은 경우 99개 이상의 파일의 정보를 가져올 수 있습니다.

    3. 다중 선택 가능한 파일 다이얼로그 띄우기

    위에서 배운 내용을 종합해 코드로 만들어보겠습니다.

    4. 여러개의 파일 정보 가져오기

    여러개의 선택된 파일 정보를 가져오기 위해서는 2가지 함수를 사용하여 구할 수 있습니다.
    이 함수들은 DoModal() 함수가 호출 이후 사용해야 합니다.

    • 아래 함수들은 CFileDialog 클래스에서 기본적으로 제공하는 함수들입니다.
      1. GetStartPosition()
      2. GetNextPathName()

    두 함수는 바로가기 파일에 대해서 값을 가져올 때 오류가 발생할 수 있으니 참고 바랍니다. GetStartPosition() 함수는 FileDialog가 가진 버퍼의 시작위치를 얻어올 수 있습니다.
    GetNextPathName() 함수는 버퍼의 담긴 여러개의 PATH 정보들을 구분해서 하나씩 얻어올 수 있습니다.
    얻어온 데이터들을 이용해 파일이름 혹은 확장명만 얻기 위해서는 따로 프로그래밍을 해야합니다.

    GetNextPathName() 함수는 더 이상 구분하여 얻어올 PATH 정보가 없다면 NULL을 반환하므로
    반복문을 이용해 하나씩 선택했던 파일에 대한 PATH 정보를 꺼내올 수 있습니다.

    얻어온 PATH 정보는 CString 타입의 데이터이며 아래 같이 PATH 정보를 가지고 있습니다.

    C:\\Temp\\Test1.txt
C:\\Temp\\Test2.txt
C:\\Temp\\Test3.txt
C:\\Temp\\Test3.txt

    GetNextPathName() 함수를 이용하여 PATH 정보를 얻어와
    CString 클래스에서 제공되는 함수들을 이용하면 파일이름 혹은 파일확장명을 구할 수 있습니다. 자체적으로 제공해주는 함수가 없으므로 따로 프로그래밍을 해야합니다.

    4. 결론

    위 내용을 정리해보겠습니다.

    • 여러개의 파일을 선택할 때
      • OFN_ALLOWMULTISELECT 플래그 적용
      • 열기모드 사용
      • 기존 FileDialog가 쓰던 버퍼는 작으므로 새로운 큰 버퍼로 변경
    • 기존 버퍼를 새로운 버퍼로 변경
      • DoModal() 함수가 호출되기 전에 작업 진행
      • CString GetBuffer(), ReleaseBuffer() 함수를 이용하여 새로운 버퍼를 만듬
      • m_ofn.lpstrFile 값을 다르게 변경하여 새로운 버퍼의 시작 위치를 가르킴
      • m_ofn.nMaxFile 값을 다르게 변경하여 버퍼 크기를 변경 (반드시 공식이용)
    • 여러개의 파일 정보를 얻기
      • GetStartPosition() 함수 반환값을 이용하여 버퍼의 시작위치를 구함
      • GetNextPathName() 함수로 버퍼의 시작위치에서 여러개의 파일 PATH 정보를 구분하여 가져옴
      • 얻어온 PATH 정보에서 파일이름 및 확장명을 얻어 오기 위해서는 따로 프로그래밍을 해야함
    • 오타 및 잘못된 내용에 대해 피드백 해주시는건 언제든 환영합니다.
