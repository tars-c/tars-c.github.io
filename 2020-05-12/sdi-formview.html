    [MFC] SDI 프로젝트 Dialog Based 처럼 사용하기

    서론

    SDI 프로젝트는 기본적으로 만들면 CView 클래스를 상속받아 만들어지기 때문에
    리소스뷰 비주얼 에디터를 이용하여 컨트롤 생성 및 배치할 수 없다.
    CView를 상속받은 경우 동적으로 컨트롤을 생성 및 배치해야한다.

    CView와 다르게 CFormView 클래스를 상속받은 경우
    Dialog를 이용한 컨트롤 생성 및 배치가 가능한다.

    이번 글에선 CFormView가 간단히 알아보고
    프로젝트를 생성할 때 CFormView를 상속받아 생성하는 법을 알아본다.
    추가로 CView를 상속받은 View를 CFormView로 상속받은 View로 바꾸는 작업도 진행한다.

    CFormView란 ?

    대화상자 리소스를 이용해 컨트롤을 배치할 수 있는 뷰이다.
    폼 기반의 MFC를 이용한 프로그램을 개발할 때 가장 기본이 되는 뷰이다.
    CSCrollView의 파생 클래스이며 스크롤 기능을 지원한다.
    CFormView를 이용하면 개발자가 양식을 만들어 다른 뷰에도 붙일 수 있다.

    다이얼로그와 연결이 되어 비주얼에디터를 사용할 수 있으므로 생산성에도 영향을 준다.

    CFormView를 상속받는 View 프로젝트 만들기

    MFC 프로젝트를 만들 때 기본적으로 CView를 상속받는다.

    다음 그림과 같이 기반 클래스를 CFormView를 변경하면 프로젝트를 생성 시 CFormView를 상속받는 View가 만들어진다.
    생각보다 너무 단순해서 놀랐다.

    CView를 상속받은 View를 CFormView로 상속받는 View로 변경하기

    솔루션 이름을 “ TE “ 로 프로젝트를 CView를 상속받은 프로젝트를 만든다. (Test 줄임말)
    Visual Studio 2017 기준으로 작성되었으며 버전이 다를 경우 올바르게 작동하지 않을 수 있다.

    1. TEView.h 를 엽니다.
    2. CView를 CFormView로 Rename(Ctrl+R+R) 합니다. (미리보기 -> 적용)
    3. 리소스뷰로 가서 Dialog 삽입 및 ID를 지정합니다. (IDD_TE)
    4. 창을 눌러 다음과 같이 속성을 변경합니다.
      • 속성 Style: Popup -> Child
      • 속성 System Menu: True -> False
      • 속성 Title Bar: True -> False
      • 속성 Border: Dialog Frame -> None
      • (요약 - ON: WS_CHILD / OFF:WS_POPUP, WS_SYSMENU, WS_BORDER)
    5. TEView.h 파일을 열어 TEView 클래스 public 멤버변수로 선언합니다.
      • enum { IDD = IDD_TE };
    6. TEView.cpp 디폴트 생성자 정의부에 멤버이니셜라이저로 CFormView 생성자를 호출한다.
      • 생성자 호출 시 필요한 매개변수는 Dialog ID 입니다.
      • : CFormView(IDD_TE)
    7. TEView.cpp 클래스 마법사를 이용해 OnInitialUpdate() 가상함수를 재정의한다.
    8. 아래 두 함수를 사용해서 뷰의 크기를 프레임과 맞추고 빌드를 한다.(딘축키: F7)
      1. GetParentFrame()->RecalcLayout();
      2. ResizeParentToFit();

    메뉴 및 툴바 삭제하기

    1. 툴바를 삭제하고 싶다면 다음과 같이 진행한다.
      • CMainFrame::OnCreate() 에서 m_wndToolBar 관련 코드를 주석처리한다.
    2. 상단 메뉴를 삭제하고 싶다면 다음과 같이 진행한다.
      • CTEApp::InitInstance() 창 초기화 작업란에 m_pMainWnd->SetMenu(NULL); 코드를 삽입한다.
