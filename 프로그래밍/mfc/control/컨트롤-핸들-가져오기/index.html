    [MFC] 컨트롤 핸들(HWND) 가져오기

    빠른 링크 (Link!)

    • 서론
    • 리소스ID를 이용해 핸들 얻기 (GetDlgItem)
    • HWND를 이용해 CWnd 인스턴스 얻기 (FromHandle)
    • CWnd 인스턴스에서 핸들(HWND) 접근
    • [F]리소스ID를 사용하여 CWnd*를 반환하는 함수
    • [F]HWND를 사용하여 CWnd*를 반환하는 함수

    서론

    MFC에는 수 많은 컨트롤이 존재합니다.
    이 컨트롤을 제어하기 위해서는 CWnd 인스턴스에 접근하거나
    핸들을 이용하는 방식이 있습니다.

    WINAPI에서는 HWND 구조체를 이용해 컨트롤을 제어했지만
    MFC에서는 조금 더 편리하게 CWnd 윈도우 클래스의 인스턴스를 접근하여 제어할 수 있습니다.

    핸들을 가져오는 방법은 여러가지 있습니다.
    그중 대표적으로 사용하는 방법에 대해 알아보겠습니다.

    리소스ID를 이용해 핸들 얻기 (GetDlgItem)

    GetDlgItem() 함수를 사용하기 위해서는 해당 컨트롤이 가진 리소스ID에 대해 알아야합니다.
    리소스ID는 숫자지만 편리하게 #define을 사용해 관리 합니다. (resource.h 파일에 명시됨)
    뷰를 여러개 사용하게 되면 해당 뷰에 컨트롤이 포함되어 있어야만 정상적으로 가져올 수 있습니다.

    this->GetDlgItem(MY_CONTROL_ID);


    GetDlgItem() 함수를 이용해 CWnd 인스턴스를 얻었으면
    형변환을 통해 해당 컨트롤 타입으로 변환하여 해당 인스턴스에 대한 함수를 사용할 수 있습니다.

    예를들어 Edit Control을 제어한다면 아래와 같이 사용할 수 있습니다.

    CEdit* pEdit = (CEdit*) this->GetDlgItem(IDC_MY_EDIT1);
CEdit->SetWindowText(_T("제어 성공!));

    HWND를 이용해 CWnd 인스턴스 얻기 (FromHandle)

    가끔 WINAPI 함수를 이용해야 한다면 핸들 구조체를(HWND) 이용할 수 밖에 없습니다.
    MFC에서는 핸들(HWND)을 이용한 접근방법보다 CWnd 인스턴스로 접근방법이 많습니다.
    그러므로 MFC는 변환하는 함수를 제공해줍니다.

    FromHandle() 함수를 이용한다면 컨트롤이 가진 핸들에 대해 CWnd* 인스턴스를 반환시켜 줍니다.
    해당 핸들의 컨트롤이 가진 CWnd 타입의 인스턴스가 없다면
    임시로 인스턴스를 생성하여 포인터를 반환해줍니다.

    임시로 생성된 인스턴스는 전역으로 사용하기에 적절하지 못합니다.
    지역변수처럼 사용하고 버려주시면 됩니다.
    FromHandle() 함수는 static 함수이므로 인스턴스가 따로 필요하지 않습니다.

    HWND editHwnd = ::CreateWindow(_T("edit"), ...);
CWnd* editWnd = CWnd::FromHandle(editHwnd);
CEdit* pEDit = (CEdit*)editWnd;

    CWnd 인스턴스에서 핸들(HWND) 접근

    CWnd 인스턴스에서는 아주 쉽게 멤버변수로 핸들(HWND)을 제공합니다.
    CWnd 클래스를 상속받은 자식 인스턴스에 대해서도 핸들(HWND)로 접근할 수 있습니다. (뷰 포함)

    this->m_hWnd;
this->GetSafeHwnd();
