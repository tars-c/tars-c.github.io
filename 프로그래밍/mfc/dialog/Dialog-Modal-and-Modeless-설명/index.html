    [MFC] Modal & Modeless Dialog : 설명

    서론

    윈도우즈에서 사용되는 Dialog는 2가지 종류로 구분됩니다.
    Modal Dialog(모달) 와 Modeless Dialog(모달리스)가 있습니다.

    둘은 서로 비슷하면서 다른 특징을 가지고 있습니다.
    윈도우즈에서 사용되는 Dialog는 부모 창이 종료되면 자동으로 종료되는 특징을 가지고 있습니다.

    CDialog 타입의 객체를 생성했을 경우 메시지 및 이벤트 처리가 상당히 복잡할 수 있습니다.
    CDialog, CDialogEx 클래스를 상속받아 자식 클래스를 만들어
    이벤트를 처리하는 방식이 편하고 좋습니다.

    1. 모달 (Modal) 설명

    Modal Dialog의 특징은 Modal Dialog가 활성화 되었을 땐
    Dialog가 종료될 때 까지 다른 창을 제어할 수 없는 제어권 독점 형태를 가지고 있습니다.

    하나의 창에서만 작업을 처리하기에 프로그래밍이 매우 간편합니다.
    사용자에게 중요한 입력을 무조건 받아야할 때 사용하면 유용합니다.
    Modal Dialog의 대표적인 예시로는 MessageBox가 있습니다.

    생성에 사용되는 방식은 정적으로 변수를 선언하거나 동적할당으로 생성이 가능합니다.
    MFC에서 DoModal() 함수를 이용하여 창을 생성합니다.
    창 닫기 및 종료는 EndDialog() 함수를 이용합니다.
    EndDialog() 함수를 호출한다고 할당했던 메모리가 자동을 해제되지는 않습니다.

    Modeless Dialog랑 다르게 창이 종료될 때 눌렀던 버튼에 해당하는 컨트롤ID 값을 반환합니다.
    반환값을 이용해 조건을 검사하여 특정 버튼을 눌렀는지에 대해 검사할 수 있습니다.

    2. 모달 (Modal) 코드

    MFC View InitDialogEx(…) 함수에서 사용 되는 코드입니다.

    InitDialog() 함수가 아닌 InitDialogEx()를 사용하는 이유는
    Based Dialog 환경에서 MainDialog가 생성된 후 바로 Dialog를 사용자에게 보여주려고 했지만,
    Modal Dialog 특징으로 창에 제어권이 독점되어 OnCreate(), OnInitDialog() 함수 처리문에서
    Modal Dialog를 생성해서 보여줘도 Main Dialog가 생성 되기 전에 Modal Dialog가 생성되서 나타나기 떄문에
    PostMessage() 함수를 이용하여 사용자 정의 메시지(WM_USER)를 보내고 처리하여 함수를 확장한 것입니다.

    Modal Dialog를 사용자에게 띄우고 지우는 간단한 예시 코드로 알아보겠습니다.
    ModalDialog는 리소스뷰에서 CDialog를 상속받아 작성된 클래스입니다.

    EndDialog() 함수가 호출되면 DoModal() 함수가 해당하는 컨트롤 ID값을 반환 후 밑에 있는 코드들이 실행됩니다.

    3. 모달리스 (Modeless) 설명

    Modeless Dialog의 특징은 창이 열리더라도
    Dialog 종료와 상관없이 다른 창을 전환하여 사용할 수 있습니다.

    일반적인 프로그램은 Modeless Dialog 방식으로 작동됩니다.
    사용자가 창을 전환할 수 있기에 다시 화면을 그리거나
    창 전환시에 특정 이벤트 및 메시지가 발생할 수 있으므로 이를 고려하는 프로그래밍을 해야합니다.
    Modeless Dialog의 예시로는 윈도우즈 메모장 찾기(Ctrl + F) Dialog입니다.

    동일한 부모 윈도우를 기준으로 Modal Dialog와 Modeless Dialog를 생성했을 때
    Modeless Dialog는 Modal Dialog 특징인 제어권 독점에 상관없이 자유롭게 전환할 수 있습니다.

    생성에 사용되는 방식은 동적할당으로만 가능하며
    MFC에서 Create() 함수를 이용하여 메모리에 할당받아
    ShowWindow() 함수를 이용하여 사용자에게 창을 보여줍니다.
    창 닫기 및 종료는 DestroyWindow() 함수를 이용합니다.
    DestroyWindow() 함수를 호출한다고 할당했던 메모리가 자동을 해제되지는 않습니다.

    4. 모달리스 (Modeless) 코드

    Modeless Dialog는 창이 종료되더라도 반환되는 ID값이 따로 없습니다.

    5. 쓸모 없는 결론

    간단한 코드와 함께 Modal과 Modeless에 대해 알아보았습니다.
    간단히 아래에 특징들을 정리해보았습니다.

    • Dialog
      • Resource View에서 상속을 받아 클래스로 확장하여 생성하는 것이 편리하고 좋음
      • EndDialog(), DestroyWindow() 함수 사용과 별개로 할당받은 메모리는 해제해야함
    • Modal
      • 제어권 독점으로 인한 창 전환불가
      • InitDialog() 에서 생성하고 보여줄 시 Main Dialog 보여지기 전에 생성됨
      • 사용자에게 중요한 것을 무조건 입력받을 때 사용 시 유용
    • Modeleess
      • Modal과 반대로 창 전환 자유롭게 가능
      • 동일한 부모를 기준으로 Modal Dialog가 보여지는 중에도 자유롭게 접근 가능
      • 찾기, 바꾸기 같은 사용자에게 편리성을 가져다주는 창이 필요할 때 사용 시 유용
    • 오타 및 잘못된 내용에 대해 피드백 해주시는건 언제든 환영합니다.
