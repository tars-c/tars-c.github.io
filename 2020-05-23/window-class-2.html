    [MFC/WINAPI] 윈도우 클래스 정보 변경하기 (DWLP 미포함)

    빠른 링크 (Link!)

    서론

    윈도우에게 속성을 추가하거나 가지고 있는 윈도우 속성을 제거하기 위해서는
    SetClassLong(), SetWindowLong() 함수를 이용해야 한다.
    무작정 속성값을 설정하는 것이 아니라
    기존 속성값을 가져(Get)와서 값 변경 후 다시 설정(Set)하도록 해야 한다.

    함수들이 비슷하므로 내용이 반복될 수 있으니 유의바란다.
    나도 반복적인 내용을 쓰고 싶지가 않다.

    SetClassLong() 함수

    윈도우 클래스(WNDCLASS)로 등록된 정보를 변경할 수 있는 함수이다.
    x86 환경에서 사용되는 함수이며 64비트 호환을 위해 SetClassLongPtr() 함수를 제공한다.

    핸들과 정보를 얻을 속성 값 그리고 변경할 값을 인자로 넘겨준다.
    속성값은 편리하게 매크로를 사용할 수 있으며
    사용되는 매크로는 함수의 앞 글자를 따서 GCL_ 로 시작된다.
    정상적으로 속성값을 변경했을 시 이전 속성값이 함수에서 반환된다.

    오류로 인한 반환값과 이전에 설정된 속성값(반환값)이 출동할 수 있다.
    GetLastError() 함수를 이용해 오류 정보를 받아볼 수 있다.

    매크로 설명
    GCW_ATOM RegisterClass() 함수가 윈도우 클래스 등록 후 리턴값
    GCL_CBCLSEXTRA 클래스 여분 메모리 양 (cbClsExtra)
    GCL_CBWNDEXTRA 윈도우 여분 메모리 양 (cbWndExtra)
    GCL_HBRBACKGROUND 배경 브러쉬 핸들 (hbrBackground)
    GCL_HCURSOR 커서 핸들 (hCursor)
    GCL_HICON 아이콘 핸들 (hIcon)
    GCL_HICONSM 작은 아이콘 핸들 (hIconSm)
    GCL_HMODULE 응용 프로그램 인스턴스 핸들 (hInstance)
    GCL_MENUNAME 뉴 리소스 문자열 (hMenu)
    GCL_STYLE 윈도우 스타일 (style)
    GCL_WNDPROC 윈도우 프로시저 (lpfnWndProc)

    Extra Class Memory 관련으로도 사용할 수 있었다.. (지금은 메모리가 넉넉하다)

    SetClassLongPtr() 함수

    SetClassLong() 함수와 크게 다를 것이 없다.
    다만, 일부 매크로 함수가 확장되었기 때문에 GCLP_ 로 시작되는 매크로를 사용해야 한다.
    Visual Stduio 가 많이 좋아져서 WinUser.h를 보면 _WIN64 매크로를 감지하여
    GCLP_ 매크로를 굳이 사용하지 않아도 자동으로 치환된다.

    x64 환경에서는 포인터 크기가 커졌기에
    지원하는 매크로 리스트를 보면 주소 및 포인터 부분이 새롭게 확장되어 지원되고 있다.

    매크로 설명
    GCLP_HBRBACKGROUND 백그라운드 브러쉬 핸들
    GCLP_HCURSOR 커서 핸들
    GCLP_HICON 아이콘 핸들
    GCLP_HICONSM 작은 아이콘 핸들
    GCLP_HMODULE 응용 프로그램 인스턴스 핸들
    GCLP_MENUNAME 메뉴 리소스 문자열
    GCLP_WNDPROC 윈도우 프로시저

    Extra Class Memory 관련으로 사용할 수 있었다.. (지금은 메모리가 넉넉하다)




    SetWindowLong() 함수

    해당 윈도우의 속성을 변경할 수 있는 함수이다.
    x86 환경에서 사용되는 함수이며 64비트 호환을 위해 SetWindowLongPtr() 함수를 제공한다.

    핸들과 정보를 얻을 속성 값 그리고 변경할 값을 인자로 넘겨준다.
    속성값은 편리하게 매크로를 사용할 수 있으며
    사용되는 매크로는 함수의 앞 글자를 따서 GWL_ 로 시작된다.
    정상적으로 속성값을 변경했을 시 이전 속성값이 함수에서 반환된다.

    오류로 인한 반환값과 이전에 설정된 속성값(반환값)이 출동할 수 있다.
    GetLastError() 함수를 이용해 오류 정보를 받아볼 수 있다.

    매크로 설명
    GWL_EXSTYLE 확장 윈도우 스타일(WS_EX…)
    GWL_HINSTANCE 응용 프로그램 인스턴스 핸들
    GWL_HWNDPARENT 부모 윈도우의 핸들
    GWL_ID 윈도우의 ID 및 식별자 (resource.h)
    GWL_STYLE 윈도우 스타일(WS_…)
    GWL_USERDATA 창과 관련된 사용자 데이터 (창을 만든 응용프로그램에서 사용하기 위한 값)
    GWL_WNDPROC 윈도우 프로시저 (CallWindowProc() 함수를 이용하여 프로시저 호출 가능)

    Extra Window Memory 관련으로 사용할 수 있었다.. (지금은 메모리가 넉넉하다)

    SetWindowLongPtr() 함수

    SetWindowLong() 함수와 크게 다를 것이 없다.
    다만, 일부 매크로 함수가 확장되었기 때문에 GWLP_ 로 시작되는 매크로를 사용해야 한다.
    Visual Stduio 가 많이 좋아져서 WinUser.h를 보면 _WIN64 매크로를 감지하여
    GWLP_ 매크로를 굳이 사용하지 않아도 자동으로 치환된다.
    캐싱문제로 속성값을 변경한 경우 SetWindowPos() 함수를 호출할 떄까지 적용되지 않는다.

    x64 환경에서는 포인터 크기가 커졌기에
    지원하는 매크로 리스트를 보면 핸들 및 포인터 부분이 새롭게 확장되어 지원되고 있다.

    인자로 넘겨준 핸들을 가지고 있는 윈도우를 가지고 있는 프로세스가
    함수 호출 프로세스와 동일한 프로세스가 아니라면 함수가 정상적으로 호출되지 않는다.

    자식창의 부모를 변경하기 위해서는
    GWLP_HWNDPARENT 속성값을 변경하기보다 SetParent() 함수를 이용한다.

    매크로 설명
    GWLP_WNDPROC 윈도우 프로시저
    GWLP_HINSTANCE 응용 프로그램 핸들
    GWLP_HWNDPARENT 부모 윈도우 핸들
    GWLP_USERDATA 사용자 데이터
    GWLP_ID 윈도우 ID (resource.h)




    정리

    1. SetClassLong()
      • WNDCLASS, WNDCLASSEX 구조체에 등록된 속성 정보를 변경시킴
      • 핸들, 변경할 속성값, 적용시킬 값을 인자로 요구함
      • x86 함수이므로 x64 환경에선 SetClassLongPtr() 함수를 사용해야함 (매크로 자동치환)
      • 리소스 정보 변경할 수 있음 (메뉴, 아이콘, 커서)
      • 백그라운드 브러쉬 핸들 변경할 수 있음
    2. SetWindowLong()
      • CreateWindow() 함수 호출 시에 사용된 윈도우 관련 속성 정보를 변경시킴
      • 핸들, 변경할 속성값, 적용시킬 값을 인자로 요구함
      • x86 함수이므로 x64 환경에선 SetWindowLongPtr() 함수를 사용해야함 (매크로 자동치환)
      • SetWindowLongPtr() 함수 호출 시 SetWindowPos() 함수를 호출해야 변경된 값이 적용됨 (캐싱)
      • SetWindowLongPtr() 함수 호출 프로세스와 창을 가지고 있는 프로세스가 같아야함
      • 확장 스타일 변경할 수 있음
      • 부모의 핸들을 변경할 수 있음 (SetParent() 함수 이용)
      • 확장 스타일(WS_EX_…) 정보를 변경할 수 있음
