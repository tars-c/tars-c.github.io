    서론

    사용자는 어떤 키를 누를지 모르는 야생마와 비슷하다.
    개발자는 프로그램을 개발하면서 사용자가 특정 키를 누른지를 구해야할 때가 있다.

    해당 키가 눌러져 있는지에 대해 확인하는 함수에 대해 알아본다.
    이때 사용되는 키코드는 Virtual Key Code 이다.
    Virtual Key Code 목록은 다음 링크에서 볼 수 있다.
    LINK: MSDN Virtual Key Code List

    GetKeyState() 함수 알아보기

    사용할 함수는 GetKeyState() 이다.
    인자로는 Virtual Key Code를 요구하고 있다.

    반환 타입은 WORD 크기(16bit)와 같은 SHORT 타입이다.
    2BYTE를 각각 쪼개거나 비트 연산을 통해 정보를 가져와 사용한다.

    반환 데이터에서 high-order bit가 1이면 해당 키를 누른 상태이며 그 외는 땐 상태를 의미한다.
    low-order bit가 1이면 키가 토글된 상태를 의미한다. 그 외는 토글되지 않은 상태를 의미한다.

    토글된 상태 값은 토글키(Toggle Key) 에만 적용되는 속성이다.
    키보드에서 토글키를 누르면 오른쪽 상단의 키보드에 불이 들어오고(1) 다시 누르면 꺼진다(0)

    • Windows 토글키는 대표적으로 3가지가 존재한다.
      • Caps Lock
      • Num Lock
      • Scroll Lock

    누른 상태 확인하기

    키를 누르고 있는지 알기 위해 high order bit와 비트 연산을 진행한다.

    16bit 데이터에서 high order bit, low order bit는 다음과 같다.
    high order bit는 1000 0000 0000 0000 을 의미하고 16진수로는 0x8000 이다.
    low   order bit는 0000 0000 0000 0001 을 의미하고 16진수로는 0x1 이다.

    ‘a’ 를 매개변수로 입력하면 정상적으로 작동하지 않는다.
    ‘a’ 는 virtual key code 목록에 존재하지 않기 때문이다. ‘A’ 를 사용하자
    virtual key code는 소문자 아스키를 지원하지 않는다는 것을 명심하자.

    a 키를 누르면 정상적으로 키가 눌린지 감지가 된다.
    Ctrl, Shift 등 다른 키를 누른 상태에서 a 키를 누르면 감지가 되지 않는 점이 있다.
