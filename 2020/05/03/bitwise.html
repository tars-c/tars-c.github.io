    서론

    저에게 XOR은 조금 친근하지 못한 연산자였는데
    한가지 쓰임을 알게게 되어 비트 연산 전체를 포스팅하게 되었습니다.
    비트 연산을 한다고 해서 피연산자를 2개 요구하는 연산자들은 결과가 바로 반영되지 않습니다.
    비트 연산도 산술연산자처럼 축약형으로 사용할 수 있습니다. (&=, |=, ^=, «=, »=)

    비트연산은 다른 연산자에 비해 처리 속도가 빠릅니다.
    공부를 할때 숫자를 막 써놔도 비트연산은 눈으로 봐도 잘 들어오지 않습니다.

    화면에 계산기를 켜두고 하나씩 실습해보면 누구나 쉽게 터득할 수 있습니다.
    비트연산 관련 프로그래밍 시 계산기를 이용하면 빠르게 작업을 처리할 수 있습니다.

    예시에서 사용되는 타입 크기는 임의 크기인 1byte입니다.
    기본 int 타입 크기는 x86 환경에서 4byte인 점을 주의하셔야 합니다.
    (자료형의 크기가 클 수록 비트가 복잡하므로 나머지 3byte는 명시 안함)

    비트는 10진수가 가진 값을 2의 n승 꼴로 표현할 수 있습니다.
    10진수와 관계지어 비트가 차지하고 있는 위치에 대해 알아야합니다.
    아래에 비트연산을 이해하기 위해서는 다음과 같은 개념을 아셔야 합니다.
    첫번째 비트: 2의 0승 = 1
    두번째 비트: 2의 1승 = 2
    세번째 비트: 2의 3승 = 4
    네번째 비트: 2의 4승 = 8
    다섯번째 비트: 2의 5승 = 16

    비트가 0000 1111 로 구성되어 있으면
    10진수 값은 15가 됩니다. (1 + 2 + 4 + 8)

    OR 연산

    OR 비트 연산자는 | 입니다.
    피연산자는 2개가 필요합니다.

    OR 비트연산은 두개의 비트 중 하나만 1이면 결과가 참(1)이 되는 것을 의미합니다.
    쉽게 이해하기 위해 변수를 만들어 연산자를 이용해 계산을 해보고 과정을 적어보겠습니다.

    int A = 5;
int B = 2;
int ans = 0;

ans = A | B;

    위 식에서 A와 B는 OR 연산자(|)를 이용해 비트 연산을 진행하게 됩니다.
    0000 0101 :: 5
    0000 0010 :: 2
    0000 0111 :: 7(결과)

    연산하는 과정을 하나씩 살펴보자면, (맨 오른쪽이 첫번째 비트)
    (5의 1번째 비트 1 | 2의 1번째 비트 0) 연산 = 1
    (5의 2번째 비트 0 | 2의 2번째 비트 1) 연산 = 1
    (5의 3번째 비트 1 | 2의 3번째 비트 0) 연산 = 1
    나머지 비트 모두 0
    연산 결과: 0000 0111 (10진수: 7)이 나오게 됩니다.

    또, OR 연산은 두개의 피연산자가 가진 값에 해당하는 비트가 중복되지 않는 선에서 더하기에도 쓰인다는 겁니다.
    5와 2를 OR 비트 연산했는데 결과값이 7이 나왔죠 ? (어이쿠! 더해졌네?)
    반대로 비트가 중복되는 경우를 보자면
    1과 3을 OR 연산하면 결과가 3이 나오게 되고 덧셈을 못하는 슬픔을 겪게 될 수도 있습니다.

    덧셈방식을 이용하여 특정 속상값을 부여할 수도 있습니다.
    0000 0000 비어 있는 비트가 있을 때
    0000 0001 10진수 1을 사용하여 OR 연산으로 1비트를 채워줄 수 있고
    0000 0010 두번째 비트도 10진수 2를 사용하여 1을 채워줄 수 있습니다.
    OR 비트 연산을 이용하면 값에 여러가지 속성값을 부여할 수 있습니다. (Flag 연산)

    WINAPI, MFC에서도 스타일처럼 여러개의 속성을 적용시킬 수 있을 때
    OR 비트 연산을 이용해 여러개의 속성을 부여하는 방식을 다음과 같이 사용합됩니다.

    cs.style = WS_VISIBLE | WS_CHILD | WS_THINFRAME;

    AND 연산

    AND 비트 연산자는 & 입니다.
    피연산자는 2개가 필요합니다.

    AND 비트 연산은 두개의 비트 중 둘다 1이어야 결과가 참(1)이 되는 것을 의미합니다.
    쉽게 이해하기 위해 변수를 만들어 연산자를 이용해 계산을 해보고 과정을 적어보겠습니다.

    int A = 5;
int B = 3;
int ans = 0;

ans = A & B;

    위 식에서 A와 B는 AND 연산자(&)를 이용해 비트 연산을 진행하게 됩니다.
    0000 0101 :: 5
    0000 0011 :: 3
    0000 0001 :: 1(결과)

    연산하는 과정을 하나씩 살펴보자면, (맨 왼쪽이 첫번째 비트)
    (5의 1번째 비트 1 & 3의 1번째 비트 1) 연산 = 1
    (5의 2번째 비트 0 & 3의 2번째 비트 1) 연산 = 0
    (5의 3번째 비트 1 & 3의 3번째 비트 0) 연산 = 0
    나머지 비트 모두 0
    연산 결과: 0000 0001 (10진수: 1)

    또, AND 연산자(&)를 이용해 속성값을 가지고 있는지에 대해 찾을 수도 있습니다.
    말로 설명하는 것보다 코드로 설명드리겠습니다.

    #include <iostream>

enum class ePermission : int {
    user = 0x01,
    group = 0x02,
    admin = 0x04
};
ePermission operator+ (const ePermission a, const ePermission b)
{
    return static_cast<ePermission>(static_cast<int>(a) + static_cast<int>(b));
}
int main()
{
    ePermission levelPermission;
    int myPermission = static_cast<int>(ePermission::user + ePermission::admin);
    if (!(myPermission & static_cast<int>(ePermission::admin)))
    {
        //invalid permission
    }
    else
    {
        // success
    }
    return 0;
}

    속성에서 관리자의 권한값은 4입니다.
    사용자가 가지고 있는 권한은 유저(1) + 관리자(4)이므로 권한값은 즉, 5입니다.
    4와 5를 AND 연산자(&)를 이용해 비트연산을 진행합니다.
    0000 0101 (10진수: 5)
    0000 0100 (10진수: 4)

    두 수가 가진 비트 중 3번째 비트가 모두 1이므로 AND 연산 결과는 1이 됩니다.
    (전 개인적으로 접근 불가나 오류 발생하는 코드가 위에 있으면 좋더라구요..)

    속성 값이 가진 비트를 뒤집어서 & 연산하면 사용자가 가지고 있는 권한을 뺏을 수 있습니다.

    myPermission &= ~ePermission::user

    NOT 연산

    NOT 비트 연산자는 ~ 입니다.
    피연산자는 1개가 필요합니다.

    NOT 비트연산은 비트를 모두 뒤집어버리는 것입니다.
    0을 1로, 1을 0으로 바꾼다 생각하시면 됩니다.

    생각보다 쓰임이 적다고 생각되는 연산자였습니다.
    알고리즈머 YunGoon님께 관련된 쓰임세를 몇가지 알아왔습니다. 감사합니다.
    쓰임1과 쓰임2에 해당하는 - 는 부호 연산자입니다. (빼기가 아닙니다)

    -~n은 n+1 꼴로 사용될 수 있다

    10진수 4를 비트로 표현하면 0000 0100 입니다.
    이를 NOT 연산하여 비트를 뒤집어 보면 1111 1011이 됩니다.
    그리고 맨 앞에 있는 -로 부호를 반전시켜주면 비트는 0000 0101이 되고 5가 되는 것입니다.
    부호를 반전시킬 때는 NOT 연산으로 비트를 모두 뒤집고 맨 오른쪽 비트에 1을 더하면 됩니다.
    1을 더하는 이유는 보수의 변환과정에서 1을 더했기 때문에 다시 더해주는 것입니다.

    0000 0100 (10진수: 4)
    1111 1011 (NOT) (10진수: -5)
    0000 0101 (부호반전) (10진수: 5)

    ~-n은 n-1 꼴로 사용할 수 있다

    쓰임1과 매우 비슷합니다. 이번엔 7로 해보겠습니다.
    10진수 7을 비트로 표현하면 0000 0111 입니다.
    2의 보수로 만드는 방법을 이용해 -7로 만들어주는 것입니다.

    음수를 표현할 때 2의 보수를 사용하는데 변환과정을 아주 간략하게 설명드리면
    양수 7이 가진 비트를 뒤집고 맨 오른쪽 비트에 1을 더해줍니다.
    -7에서 NOT 연산을 하여 비트를 뒤집어주면 6이 됩니다.

    0000 0111 (10진수: 7)
    1111 1001 (부호반전) (10진수: -7)
    0000 0110 (NOT) (10진수: 6)

    EOF라면 루프 탈출에 사용할 수 있다

    수 많은 컴파일러가 EOF를 -1로 정의합니다.
    -1 값을 가지고 반복문을 사용한다면 NOT 연산을 통해 비트를 뒤집습니다.
    -1 값이 0 으로 변경되고 if, while, for 같은 조건문에서 활용될 수 있습니다.

    1111 1111 (10진수: -1)
    0000 0000 (NOT) (10진수: 0)

    이는 while(~scanf()) { } 처럼 응용할 수 있습니다.

    XOR 연산

    XOR 비트 연산자는 ^ 입니다.
    피연산자는 2개가 필요합니다.

    XOR 비트연산은 OR과 비슷하면서도 전혀 다릅니다.
    두 수중 비트를 하나씩 비교하여 둘다 1이거나 0일 떄 0을 반환합니다.
    서로 같은 값이면 0, 서로 같지 않으면 1을 반환한다고 생가하시면 됩니다.
    쉽게 이해하기 위해 변수를 만들어 연산자를 이용해 계산을 해보고 과정을 적어보겠습니다.

    숫자 7과 3을 예시로 들어보겠습니다.

    int A = 7;
int B = 3;
int ans = 0;

ans = A ^ B;

    위 식에서 A와 B는 XOR 연산자(^)를 이용해 비트 연산을 진행하게 됩니다.
    0000 0111 :: 7
    0000 0011 :: 3
    0000 0100 :: 4(결과)

    연산하는 과정을 하나씩 살펴보자면, (맨 왼쪽이 첫번째 비트)
    (7의 1번째 비트 1 & 3의 1번째 비트 1) 연산 = 0
    (7의 2번째 비트 1 & 3의 2번째 비트 1) 연산 = 0
    (7의 3번째 비트 1 & 3의 3번째 비트 0) 연산 = 1
    나머지 비트 모두 0 또는 1로 같으므로 0

    XOR은 알면 알 수록 신기한 녀석입니다. (누가 발견한거지?)
    XOR은 이미지 연산이나 교체 알고리즘에 사용됩니다.
    우연히 알파벳을 빠르게 교체하는 문제를 풀면서 새로운 사용 방법을 알게되었습니다.
    이 방법도 알고리즈머 YunGoon님께서 알려주셨습니다. (깊게 생각했으면 스스로 알았을까?)


    위 그림처럼 10진수 32는 비트를 1개만 가지고 있습니다.
    반대로 31은 32가 가지고 있는 비트를 제외한 나머지 하위 비트를 모두 가지고 있습니다.
    (이미 알고있는 정보인데도 뭔가 다시 깨달은 느낌이랄까요 ?)

    비트가 가득찬 경우

    10진수 31과 5를 XOR 연산해보겠습니다.
    0001 1111 :: 31
    0000 0101 :: 5
    0001 1010 :: 26(결과)

    자세한 연산과정은 생략하겠습니다.
    재밌는 사실은 31에서 5를 뺀 값인 26이 나온다는 겁니다.
    주의할 점은 31과 31을 연산하면 0이 나오니 주의해야합니다.

    비트가 비어 있는 경우

    10진수 32와 5를 XOR 연산해보겠습니다.
    0010 0000 :: 32
    0000 0101 :: 5
    0010 0101 :: 37

    자세한 연산과정은 생략하겠습니다.
    재밌는 사실은 32에서 5를 더한 값인 37이 나온다는 겁니다.
    32가 가지고 있는 비트 중 연산하려는 5의 비트가 비어있는 경우 덧셈을 하게 됩니다.

    위 방식을 응용한다면 영어 대소문자를 변환하는 작업에 처리속도를 향상시킬 수 있습니다.

    SHIFT 연산

    SHIFT 비트 연산자는 <<, >> 입니다.
    피연산자는 2개가 필요합니다.

    왼쪽 피연산자에는 비트를 조작할 데이터를 적고
    오른쪽 피연산자에는 몇칸을 밀 것인지에 대해 적습니다.

    SHIFT 연산은 비트를 오른쪽 혹은 왼쪽으로 이동할 때 사용합니다.
    왼쪽으로 비트가 밀려서 범위를 초과하면 값이 사라지게 됩니다.

    양수는 최상위 부호비트가 0이므로 어떻게 밀어도 상관이 없습니다.
    음수인 경우에 SHIFT 연산할 경우 최상위 비트(MSB, 부호비트)도 같이 밀리기 때문에 주의해야합니다.

    오른쪽으로 « 밀어서 범위 초과로 값이 이상한 경우 오버플로우(overflow)
    왼쪽으로 » 밀어서 범위 초과로 값이 이상한 경우 언더플로우(underflow)라 합니다.
    SHIFT 연산은 단독으로 사용되기보다 | 및 & 연산과 함께 사용되는 편입니다.

    SHIFT « (오른쪽으로 시프트)

    31 << 1;

    31 이 가지고 있는 전체 비트를 오른쪽으로 1칸 밀게 됩니다.
    오른쪽으로 비트가 밀리면 빈자리는 0으로 채워집니다.

    0001 1111 :: 31
    0011 1110 :: 62(결과)
    결과를 보면 알 수 있듯이 오른쪽으로 비트를 1칸 밀 수록 2배 커집니다.

    그럼 한번에 2칸을 밀면 2배 커질까요 ? 아닙니다.
    4배 커지게 되는데 이는 오른쪽으로 비트를 밀 수록 2의 n승만큼 커지는 것입니다.

    31 « 3을 계산해봅시다.
    2의 3승은 8 즉, 31x8 값인 248이 결과로 산출됩니다.

    SHIFT » (왼쪽으로 시프트)

    79 >> 2;

    79 가 가지고 있는 전체 비트를 왼쪽으로 2칸 밀게 됩니다.
    왼쪽으로 비트가 밀려서 자료형 범위를 초과하게 되면 값이 사라지게 됩니다.

    0100 1111 :: 79
    0001 0011 :: 19(결과)

    결과를 보면 알 수 있듯이 왼쪽으로 비트를 1칸 밀 수록 2배 작아집니다.

    SHIFT, AND를 이용한 색상값 마스크

    SHIFT 연산은 컬러값에서 마스크 연산에 사용 됩니다.
    색상은 RGB로 표현하는데 0~255 값으로 구성되어 있습니다.

    RGB 하얀색 색상값을 가진 값은 0xFF(R) FF(G) FF(B)가 됩니다.
    16진수 기준 15는 0xF입니다.
    10진수 255는 16진수로 0xFF가 됩니다.
    FF를 2진수로 표현하면 1111 1111 입니다.

    10진수 65,535는 16진수 0xFFFF가 됩니다.
    10진수 16,777,215는 16진수 0xFFFFFF가 됩니다.

    0xFFEE11 컬러값에서 RGB중 R을 나타내는 FF 부분을 뽑아 내려면
    오른쪽으로 8칸씩 2번 밀면됩니다. (총 16번)

    비트로 표현해보겠습니다.
    0001 0001 :: 11
    1110 1110 :: EE
    1111 1111 :: FF 0xFFEE11 » 16 0xFF


    처음 8칸을 오른쪽으로 밀면 가장 하위 비트에 속하는 11은 사라지게 됩니다.
    다시 8칸을 오른쪽으로 밀면 하위 비트에 속하는 EE는 사라지게 됩니다.
    색상값을 비트를 밀어서 찾아낼 수 있습니다.

    그럼 만약 ? EE 값을 뽑기 위해선 어떤 방법을 취해야할까요 ?
    11은 찾는 값이 아니므로 8칸을 밀어서 버려줍니다.
    0xFFEE 값 중 EE 값을 추출하기 위해서는 0xFF 값과 AND 연산을 진행하면 됩니다.

    이를 비트로 표현해보겠습니다. 8byte마다 구분은 / 로 하겠습니다.
    1111 1111 / 1110 1110 :: FFEE
    0000 0000 / 1111 1111 :: FF
    0000 0000 / 1110 1110 :: EE(결과)

    지금까지 비트연산과 쓰임세를 알아보는 유익한 시간이었습니다.
