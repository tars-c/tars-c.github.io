    서론

    이 글에서는 CFileDialog가 제공하는 함수 및 변수를 이용한 실습은 다루지 않습니다.

    MFC에서 사용자는 CFileDialog를 이용해 편리하게 파일을 저장하고 열 수 있습니다.
    흔히 사용되는 메모장 프로그램에서도 FileDialog를 볼 수 있습니다.
    CFileDialog는 Modal Dialog 방식을 이용해
    DoModa() 함수를 호출하여 사용자에게 보여주고 사용합니다. 인스턴스를 생성 후 EndDialog() 함수를 호출하여 객체를 소멸하지 않아도 자동으로 처리됩니다.

    CFileDialog 인스턴스를 생성하여 사용자에게 FileDialog를 보여주고
    파일을 선택하면 파일에 대한 정보를 CFileDialog 인스턴스로 가져오게 됩니다.
    인스턴스 멤버변수에 접근하여 관련된 정보를 사용할 수 있습니다.

    함수 매개변수를 살펴봅시다.

    CFileDialog 매개변수

    • CFileDialog 인스턴스를 생성하기 위해서는 8가지 매개변수가 필요합니다.
      1. FileDialog 파일 열기 및 저장 모드
      2. 기본 확장자 (저장 시 사용됨)
      3. 기본 파일명
      4. 열기모드 flags
      5. 필터
      6. 부모 윈도우
      7. OPENFILENAME 구조체의 크기입니다.
      8. 비스타 스타일

    하나씩 풀어봅시다.

    1. 파일을 여는건 TRUE, 저장하는건 FALSE입니다.
    2. 기본확장자는 저장시에만 사용되며 필터링이 모든 파일(*.*)인 상태에서 사용가능합니다.
        단순히 파일을 저장하는 것이라면 NULL을 추천드립니다.
    3. 기본파일명은 저장 및 열기할 때 아래 ‘파일 이름’ 란에
      기본적으로 적혀있는 파일 이름을 나타냅니다.
    4. OPENFILENAME 구조체에 해당하는 플래그 값을 의미합니다.
    5. 사용자가 해당 확장명을 가진 파일만 볼 수 있게 해주는 파일 확장명 필터입니다.
    6. 부모 윈도우를 의미합니다.
        부모 윈도우를 지정하는 이유는 부모창에서 Dialog를 띄워서 보여주기 때문입니다.
    7. OPENFILENAME 구조체의 크기를 이용하여 적절한 종류의 대화상자를 결정하게 됩니다. (기본값 사용)
    8. TRUE는 비스타 윈도우 스타일을 사용하고 FALSE는 윈도우10 스타일을 사용합니다.
        Windows10 기준, Visual Studio 2008 이상

    CFileDialog 확장명을 필터링 - 필터설명

    FileDialog는 확장명에 대해서 필터링 할 수 있습니다.
    여러 확장명을 필터링 할 수 있고, FileDialog 필터 리스트에 여러개의 필터를 추가할 수 있습니다.

    필터(Filter)는 필터설명,필터확장명 으로 구성되어 있습니다.
    각 구성 요소들은 | 를 사용하여 구분하게 됩니다.
    필터에 대한 텍스트 틀은 필터설명 | 필터확장명; ||입니다.
    마치 문자열 끝에 널값이 들어가듯이 필터 끝에는 || 문자가 2개 들어갑니다.

    필터설명은 정말 필터에 대한 설명을 의미하기 때문에 아무거나 입력하셔도 됩니다.
    필터설명 요소 맨 뒤에는 필터링 할 파일확장명을 써주는 것이 좋습니다.
    파일확장명을 쓰지 않을 경우 자동으로 구성요소 필터확장명에 적은적은 내용이 소괄호(,)로 감싸서
    필터설명 끝에 덧붙여 FileDialog 필터 리스트에 나타납니다.

    예시:
	case1 (필터설명에 파일 확장명 미기입):
		필터설명: Binary Files
		필터확장명: *.bin;*.hex;
		출력되는 필터명: Binary Files(*.bin;*.hex;)

	case2 (파일 확장명 기입):
		필터설명: Binary Files (*.bin, *.hex)
		필터확장명: *.bin; *.hex;
		필터 리스트: Binary Files (*.bin, *.hex)

    필터설명은 보통 All Files (*.*) 처럼 알아보기 쉽게
    간단한 필터에 대한 설명과 함께 확장명을 써주는 것이 좋습니다.

    CFileDialog 확장명을 필터링 - 필터확장명

    필터확장명은 필터할 파일 확장명을 의미합니다.

    필터설명에 확장명을 적더라도 두번째 요소에 해당하는
    파일확장명을 어떻게 적냐에 따라서 FileDialog에서 파일을 필터링하게 되는 점을 유의하셔야 합니다.
    필터확장명은 파일확장명을 *.bin; *.txt; 처럼 입력하면 파일을 필터링할 수 있습니다.
    필터링할 확장명을 입력하고 반드시 세미콜론(;)을 끝에 써줘야합니다.
    세미콜론(;)을 써주지 않은 경우 필터가 제대로 작동하지 않을 수 있습니다.
    만약 1개의 필터에서 3개의 확장명을 필터링한다면 필요한 세미콜론(;)은 3개입니다.

    확장명을 필터링하는 예시를 적어보겠습니다.

    // 1개의 필터에서 모든 파일 확장명을 필터링해보겠습니다.
All Files (*.*) | *.*; ||

    // 1개의 필터에서 2개의 파일 확장명을 필터링해보겠습니다.
Text or Image Files (*.txt, *.jpg) | *.txt;*.jpg; ||

    // 2개의 필터에서 각각 1개의 파일 확장명을 필터링해보겠습니다.
Text Files (*.txt) | *.txt; | IMG Files | (*.jpg) | *.jpg; ||

    위 필터를 사용하게 되면 텍스트(*.txt) 그리고 이미지(*.jpg) 파일이 입력한 필터에 맞게 필터링 되어
    폴더를 제외하고 해당하는 확장명을 가진 파일에 대해서만 FileDialog 필터 리스트에서 보여지게 됩니다.
