    [삽질기] Shift, Ctrl, Alt 보조키 반복키 인식 문제

    서론

    CTRL, SHIFT, ALT 키는 윈도우즈에서 보조키라고 불린다.
    보조키도 일반 키보드와 똑같이 꾹 누르고 있을 때 계속 메시지가 처리되어야한다.

    누군가 말하기 전까지 나는 알지 못했다.
    어떤 키보드를 누르고 있을 땐 메시지가 계속 처리되야 한다는 걸

    WM_KEYDOWN 메시지를 처리하는 OnKeyDown 핸들러에서
    키가 눌렀을 때를 감지하기 위해서 고군분투하며 5시간을 허비했다.

    더 늦게 말씀해주셨다면 하루 이상 소비하지 않았을까…
    도움을 주신분들께 모두 정말 감사합니다.

    문제점

    VK_BACK은 백스페이스의 가상 키코드이다.
    VK_SPACE는 스페이스바의 가상 키코드이다.

    백스페이스 키와 스페이스바 키를 누르고 있을 때에는 계속해서 메시지가 발생한다.
    이를 반복키라고 표현하겠다. 그리고 위 두 키에는 반복키가 적용이 되고 있다. 다른 키를 눌러도 반복키는 적용 되어야한다.

    하지만 SHIFT 키를 누를 시 메시지가 한번 밖에 발생되지 않고 있다.
    즉, 반복키가 적용되지 않는다.

    더 심각한건 누를 때가 아니라 키보드에서 손을 떌 때 핸들러가 호출되고 있었다.
    그리고 난 위 방식처럼 작동되는 것이 윈도우즈 10에서 정해진 약속이다 라고 생각했다.

    Shift, Ctrl 키는 특수한 키니까… 당연히 이렇게 작동이 되겠지 라고
    계속.. 계속.. 이상한 약속을 하며 나는 개발에 집중했다.

    문제 해결 시도

    개발을 진행해 나아가면서
    눌렀을 때를 감지해야하므로 키 인식을 해야하기 때문에
    GetAsyncKeyState() GetKeyState() Accelerator() 등 모두 사용해보고
    조건문도 바꾸고 다른 함수도 찾아보고 stackoverflow 관련 이슈도 찾으면서 별짓을 다해봤다.

    근데 아무리 해봐도 키를 누를 때를 감지하지 못했다.
    아직 부족한 지식으로 검색을 통해 후킹을 시도해보기도 해봤다.
    하지만 문제를 해결하기에는 역부족이었다.

    도움 요청

    혹시 몰라서 카카오톡 개발방에 도움을 부탁드렸다.

    방에서는 여러방법들을 제안해주셨지만 해결책으론 부족했다.
    지쳐있는 중에 윈도우 잡부 라는 닉네임을 가지신 분께서 정말 큰 도움을 주셨다.
    윈도우 잡부: (직접 테스트 코드를 보여주시면서) 저는 정상 작동 하는데요
    나: (당황하며) 혹시 SHIFT 키를 누르고 계실 때 계속해서 메시지가 호출되나요 ?
    윈도우 잡부: 네 아주 잘되는데요.

    윈도우즈10에서 작동 되는지 확인하기 위해 아버지가 사용중이신 컴퓨터를 잠시 빌려 테스트해봤다.
    확인해보니까.. SHIFT 키를 누르고 있으면 메시지 핸들러가 계속 호출되고 있는 것이다.
    충격 그자체였다.

    원인1: 절전모드

    결국에 내가 사용중인 운영체제 환경이 잘못된 것이 맞았다.
    지금까지 윈도우 환경이랑 100% 같은 줄 알았는데 아니었다.

    근본적인 문제는 윈도우즈10을 구동시키는 가상머신이 문제였다.
    키보드 설정하는 부분에서 게임으로 최적화 하여 반응속도를 빠르게 해놨어야 했는데
    배터리 아낀다는 걸 중요하게 여겨서 모두 최대 절전을 해버린 것이다.
    이정도로 키보드 반응속도 까지 조절할 줄이야… 가상먼신 기술에 감탄한다

    옵션을 게임 최적화로 바꾸어 보고 재부팅 후 테스트해봤다.
    키보드를 누르자마자 메시지가 핸들러가 호출된다.
    문제는 한가지 더 남아있었다.

    원인2: 키보드 호환

    첫번째 문제를 해결했지만
    또 다른 문제점은 누르고 있을 때 메시지가 반복되지 않는다는 것이다.
    즉, 반복키가 적용되지 않고 있다.

    눌렀을 때에만 감지해도 프로그램 개발하는데 문제는 없을 것 같지만
    윈도우즈10 사용자와 똑같은 환경에서 개발해야하는게 당연한거 아닌가라는 생각이 들었다.

    구글에서 찾아봤는데 키보드 호환 문제로 일부 노트북에서 반복키가 설정되지 않는다는 것이다.
    윈도우즈에서는 키보드 호환 문제를 필터키라는 기능으로 개선 방안을 마련해준다.

    그리고 필터키를 사용함으로 써 키가 눌리고 있을 대 키보드의 반응 속도를 설정할 수 있었고
    키를 누르고 있으면 계속해서 메시지가 처리가 되지 않았던 문제를 해결했다.

    해결 방안

    절전모드 해제

    1. 키보드 속도 개선
      • 윈도우즈10 구성 -> 마우스 및 키보드로 이동
        • 키보드: [✔]게임에 적합한 최적화
    2. 단축키 충돌 방지
      • 앱 환경설정(⌘ + ,) -> 단축키로 -> Windows10 클릭
        • ⇧ 에서 SHIFT로 등록
    3. 단축키 속도 개선
      • 윈도우즈10 구성 -> 부트 순서 -> 외부 부팅장치 -> 고급설정으로 이동
        • 부트 플래그 추가: devices.kbd.end_delay=10

    참고 사이트: Parallels: Shift key is not working inside Windows computer connected by RDP

    키 반복 반응속도 변경

    1. 필터키 켜 ㅇ기
      • 제어판(큰 아이콘) -> 접근성 센터 -> 키보드를 사용하기 쉽게 설정 -> 필터 키 설정
        • [✔]필터 키 켜기
      • 바로 가기 키
        • 모두 체크 해제
      • 필터 옵션
        • [✔]반복 키 및 느린켜기
      • 기타 설정
        • 모두 체크 해제
    2. 반복 키 및 느린키 설정으로 이동하여 다음과 같이 설정한다.
      • [✔]키 반복 속도 늦춤
        • 0.3초
        • 0.3초
    3. 레지스트리를 이용한 키보드 반응속도 변경
      • 기본적으로 설정된 반응 속도는 조금 느리다.
      • 빠르게 변경하려면 레지스트리 수정을 해야한다.
      • 레지스트리 편집기(regedit)를 열어 다음과 같이 설정한다.
        • 컴퓨터\HKEY_CURRENT_USER\Control Panel\Accessibility\Keyboard Response 이동
        • AutoRepeatDelay 200 ~ 250
        • AutoRepeatRate: 10 ~ 50
        • Last Valid Delay: 200 ~ 250 (10진수)
        • Last Valid Repeat: 10 ~ 50 (10진수)

    참고 사이트: 뱃푸와 레몬 티스토리

    주의사항은 접근성 센터에서 설정을 덮었으면 다시 레지스트리를 수정해야한다.
    레지스트리를 변경했다면 재부팅은 필수다.
